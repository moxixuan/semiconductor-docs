# 🔗 服务器访问指南

## 🌐 访问地址

### 服务器信息
- **服务器IP**: 172.17.0.1
- **端口**: 3000
- **访问地址**: http://172.17.0.1:3000

### 方式1：使用HTTP服务器（推荐，避免Node版本问题）

```bash
# 1. 先检查是否有预构建版本
ls -la build/ | head -10

# 如果build目录存在，运行HTTP服务器脚本
cd ~/semiconductor-docs
./start-server.sh

# 如果build不存在，先构建（可能失败）
npm run build

# 然后运行HTTP服务器
./start-server.sh
```

### 方式2：SSH隧道（如需要）

```bash
# 在本地电脑执行
ssh -L 3000:localhost:3000 user@server

# 然后在本地访问 http://localhost:3000
```

### 方式3：端口转发

如果您无法直接访问172.17.0.1，可以：

#### 方法A：端口转发
```bash
# 在本地电脑（需要SSH访问服务器）
ssh -L 8080:server:3000 user@server
# 然后访问 http://localhost:8080
```

#### 方法B：X11转发
```bash
ssh -X user@server
cd ~/semiconductor-docs
./start-server.sh
```

## 🔧 故障排查

### 问题：端口被占用

```bash
# 检查端口
netstat -tuln | grep :3000

# 或使用其他端口
python3 -m http.server 3001 --bind 0.0.0.0
```

### 问题：无法访问

**可能原因**：
1. 防火墙阻止
2. 服务器网络配置
3. IP地址变化

**解决方法**：
```bash
# 检查监听状态
netstat -tuln | grep :3000

# 检查防火墙
sudo ufw status  # Ubuntu
sudo firewall-cmd --list-all  # CentOS

# 重启服务器网络（如有需要）
sudo systemctl restart network
```

### 问题：静态文件不存在

```bash
# 检查build目录
ls -la build/ | head -5

# 如果为空，说明需要先构建
# 但由于Node版本限制，构建可能失败
# 建议在本地完成构建
```
